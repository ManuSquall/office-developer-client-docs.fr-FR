## <a name="two-ways-to-set-properties"></a><span data-ttu-id="79954-108">Deux manières de définir des propriétés</span><span class="sxs-lookup"><span data-stu-id="79954-108">Two ways to set properties</span></span>

<span data-ttu-id="79954-p102">La raison d'être de la boîte de dialogue des propriétés personnalisée est que toutes les applications qui utilisent des contrôles ActiveX ne fournissent pas une feuille des propriétés comme Microsoft Access. <span data-ttu-id="79954-111">Pour certaines propriétés de contrôle ActiveX, vous pouvez choisir un des deux endroits suivants pour définir la propriété :</span><span class="sxs-lookup"><span data-stu-id="79954-111">For some ActiveX control properties, you can choose either of these two locations to set the property:</span></span>

- <span data-ttu-id="79954-112">La feuille des propriétés Microsoft Access.</span><span class="sxs-lookup"><span data-stu-id="79954-112">The Microsoft Access property sheet.</span></span>
- <span data-ttu-id="79954-113">La boîte de dialogue des propriétés personnalisée du contrôle ActiveX.</span><span class="sxs-lookup"><span data-stu-id="79954-113">The ActiveX control's custom properties dialog box.</span></span>

<span data-ttu-id="79954-p103">Dans certains cas, la boîte de dialogue des propriétés personnalisée est la seule manière de définir une propriété en mode Création. For example, the **GridFont** property for the Calendar control has a number of arguments; you can't set more than one argument per property in the Microsoft Access property sheet.</span></span>
