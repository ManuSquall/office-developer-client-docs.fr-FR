<span data-ttu-id="7553f-105">Lorsqu'un fournisseur de services prend en charge les propriétés nommées, chaque ensemble d'identificateurs et de paires de noms est appelé mappage.</span><span class="sxs-lookup"><span data-stu-id="7553f-105">When a service provider supports named properties, each set of identifier and name pairs is referred to as a mapping.</span></span> <span data-ttu-id="7553f-106">Les fournisseurs de services peuvent prendre en charge un ou plusieurs mappages.</span><span class="sxs-lookup"><span data-stu-id="7553f-106">Service providers can support one mapping or several.</span></span> <span data-ttu-id="7553f-107">En d'autres, un fournisseur de banque de messages, par exemple, peut implémenter les méthodes **GetIDsFromNames** et **GetNamesFromIDs** data-stu-id="7553f-107">That is, one message store provider, for example, can implement the **GetIDsFromNames** and **GetNamesFromIDs** methods for all of its message, folder, and message store objects to work with a single list of names and their corresponding identifiers.</span></span> <span data-ttu-id="7553f-108">Un autre fournisseur de banque de messages peut avoir une liste pour chaque fournisseurs de banques de messages qui utilisent un mappage unique pour chaque message ne doivent pas autoriser les propriétés nommées à apparaître dans leurs tables de contenu de dossier, car pour class="sxs-lookup"><span data-stu-id="7553f-110">MAPI recommends that providers keep it simple and operate with a single list for all of their objects including tables.</span></span> 
  
<span data-ttu-id="7553f-111">Pour chaque mappage, les fournisseurs de services doivent fournir une signature de mappage.</span><span class="sxs-lookup"><span data-stu-id="7553f-111">For every mapping, service providers must supply a mapping signature.</span></span> <span data-ttu-id="7553f-112">Une signature de mappage est une valeur binaire, généralement un GUID, qui identifie de manière unique un ensemble d'identificateurs de propriétés et leurs noms correspondants.</span><span class="sxs-lookup"><span data-stu-id="7553f-112">A mapping signature is a binary value, usually a GUID, that uniquely identifies a set of property identifiers and their corresponding names.</span></span> <span data-ttu-id="7553f-113">Les signatures de mappage sont stockées dans la propriété **PR_MAPPING_SIGNATURE** ([PidTagMappingSignature](pidtagmappingsignature-canonical-property.md)) d'un objet.</span><span class="sxs-lookup"><span data-stu-id="7553f-113">Mapping signatures are stored in an object's **PR_MAPPING_SIGNATURE** ([PidTagMappingSignature](pidtagmappingsignature-canonical-property.md)) property.</span></span> <span data-ttu-id="7553f-114">Les fournisseurs de services doivent modifier la valeur de leur propriété **PR_MAPPING_SIGNATURE** chaque fois qu'une modification est apportée au mappage qu'elle représente.</span><span class="sxs-lookup"><span data-stu-id="7553f-114">Service providers must change the value for their **PR_MAPPING_SIGNATURE** property whenever a change is made to the mapping that it represents.</span></span> <span data-ttu-id="7553f-115">Par exemple, **PR_MAPPING_SIGNATURE** doit être mis à jour si un nouvel identificateur est affecté à un nom ou si une paire nom/identificateur est ajoutée.</span><span class="sxs-lookup"><span data-stu-id="7553f-115">For example, **PR_MAPPING_SIGNATURE** must be updated if a new identifier is assigned to a name or a new name and identifier pair is added.</span></span> 
  
<span data-ttu-id="7553f-116">Les clients qui travaillent avec les propriétés nommées des objets utilisent les propriétés **PR_MAPPING_SIGNATURE** des objets dans les opérations de comparaison et de data-stu-id="7553f-117">To compare named property identifiers belonging to two objects, clients not using mapping signatures must call [IMAPIProp::GetNamesFromIDs](imapiprop-getnamesfromids.md) on both objects to retrieve the names for each of the identifiers.</span></span> <span data-ttu-id="7553f-118">L'utilisation des signatures de mappage d'objets peut rendre cet appel inutile.</span><span class="sxs-lookup"><span data-stu-id="7553f-118">Using the mapping signatures of objects can render this call unnecessary.</span></span> <span data-ttu-id="7553f-119">Lorsque deux objets ont la même **PR_MAPPING_SIGNATURE** properties, they use the same mapping.</span></span> <span data-ttu-id="7553f-120">Les identificateurs qui utilisent le même mappage peuvent être comparés de services qui implémentent [IMAPIProp:: CopyTo](imapiprop-copyto.md) et [IMAPIProp:: CopyProps](imapiprop-copyprops.md) peuvent également tirer parti de la signature de mappage d'un objet.</span><span class="sxs-lookup"><span data-stu-id="7553f-121">Service providers that implement [IMAPIProp::CopyTo](imapiprop-copyto.md) and [IMAPIProp::CopyProps](imapiprop-copyprops.md) can also take advantage of an object's mapping signature.</span></span> <span data-ttu-id="7553f-122">Lors de la copie de propriétés nommées entre des objets, les fournisseurs de services peuvent éviter l'étape de conversion lorsque les objets source et de destination ont la même signature de mappage.</span><span class="sxs-lookup"><span data-stu-id="7553f-122">When copying named properties between objects, service providers can avoid the conversion step when the source and destination objects have the same mapping signature.</span></span>
