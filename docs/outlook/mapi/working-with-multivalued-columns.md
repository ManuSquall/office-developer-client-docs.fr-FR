<span data-ttu-id="a5f5d-105">Une colonne à valeurs multiples contient les données d’une propriété à valeurs multiples, qui est une propriété qui possède un tableau de valeurs du type de base au lieu d’une valeur unique.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-105">A multivalued column contains the data of a multivalued property, which is a property that has an array of values of the base type instead of a single value.</span></span> <span data-ttu-id="a5f5d-106">Étant donné qu’aucune des tables n’inclut de propriétés à valeurs multiples dans leurs jeux de colonnes par défaut, les propriétés à valeurs multiples ne sont incluses dans une table que si l’utilisateur de la table le demande.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-106">Because none of the tables include multivalued properties in their default column sets, multivalued properties are included in a table only if the user of the table requests it.</span></span> 
  
<span data-ttu-id="a5f5d-107">Les colonnes à valeurs multiples peuvent être affichées dans des tableaux :</span><span class="sxs-lookup"><span data-stu-id="a5f5d-107">Multivalued columns can be displayed in tables:</span></span>

    - <span data-ttu-id="a5f5d-110">Ou -</span><span class="sxs-lookup"><span data-stu-id="a5f5d-110">Or -</span></span>
    
- <span data-ttu-id="a5f5d-111">Dans une série de lignes, avec une ligne pour chacune des valeurs de propriété.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-111">In a series of rows, with one row for each of the property values.</span></span> <span data-ttu-id="a5f5d-112">Chaque valeur unique apparaît dans la colonne dans sa propre ligne avec autant de lignes que de valeurs dans la propriété à valeurs multiples.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-112">Each unique value appears in the column in its own row with there being as many rows as there are values in the multivalued property.</span></span> <span data-ttu-id="a5f5d-113">Chaque ligne a une valeur unique pour la **propriété PR_INSTANCE_KEY** ([PidTagInstanceKey](pidtaginstancekey-canonical-property.md)), mais les mêmes valeurs pour les autres colonnes.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-113">Each row has a unique value for the **PR_INSTANCE_KEY** ([PidTagInstanceKey](pidtaginstancekey-canonical-property.md)) property, but the same values for the other columns.</span></span> <span data-ttu-id="a5f5d-114">Si une ligne contient plusieurs colonnes avec plusieurs valeurs, par exemple, deux colonnes avec des valeurs  _M_ et  _N_ respectivement, les instances  _M \* N_ de la ligne apparaissent dans le tableau.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-114">If a row contains more than one column with multiple values, for example, two columns with  _M_ and  _N_ values respectively, then  _M\*N_ instances of the row appear in the table.</span></span> 
    
<span data-ttu-id="a5f5d-115">Un utilisateur de table demande le type d’affichage non par nature en appelant la méthode [IMAPITable::SetColumns](imapitable-setcolumns.md) avec l’indicateur MVI_FLAG  <span data-ttu-id="a5f5d-120">**Restrict,** cependant, ne développe pas la colonne à valeurs multiples en lignes calculées supplémentaires.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-120">**Restrict**, however, does not expand the multivalued column into additional computed rows.</span></span> <span data-ttu-id="a5f5d-121">Une colonne à valeurs multiples avec le MVI_FLAG indique au fournisseur de services d’utiliser cette colonne pour limiter le tableau.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-121">A multivalued column with the MVI_FLAG set instructs the service provider to use that column in restricting the table.</span></span> <span data-ttu-id="a5f5d-122">S’il existe une valeur de propriété dans la restriction, il doit s’agit d’une balise de propriété de valeur unique identique à celle qui serait renvoyée par [IMAPITable::QueryRows](imapitable-queryrows.md) pour la colonne.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-122">If there is a property value in the restriction, it must be a single value property tag identical to the one that would be returned by [IMAPITable::QueryRows](imapitable-queryrows.md) for the column.</span></span> 
  
<span data-ttu-id="a5f5d-123">Les implémenteurs de tableau sont requis uniquement pour prendre en charge le type d’affichage par défaut et peuvent renvoyer la valeur MAPI_E_TOO_COMPLEX lorsqu’un appelant demande l’autre alternative.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-123">Table implementers are only required to support the default type of display and can return the MAPI_E_TOO_COMPLEX value when a caller requests the other alternative.</span></span> <span data-ttu-id="a5f5d-124">La possibilité de prendre en charge ces deux types d’affichage est particulièrement importante pour les fournisseurs de magasins de messages qui implémentent des tables de contenu de dossiers.</span><span class="sxs-lookup"><span data-stu-id="a5f5d-124">The ability to support both types of display is most important for message store providers implementing folder contents tables.</span></span>
