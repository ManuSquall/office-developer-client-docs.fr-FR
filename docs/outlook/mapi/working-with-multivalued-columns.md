<span data-ttu-id="0840c-105">Une colonne à valeurs multiples contient les données d'une propriété à valeurs multiples, qui est une propriété qui a un tableau de valeurs du type de base au lieu d'une valeur unique.</span><span class="sxs-lookup"><span data-stu-id="0840c-105">A multivalued column contains the data of a multivalued property, which is a property that has an array of values of the base type instead of a single value.</span></span> <span data-ttu-id="0840c-106">Comme aucune des tables n'inclut de propriétés à valeurs multiples dans leurs jeux de colonnes par défaut, les propriétés à valeurs multiples ne sont incluses dans un tableau que si l'utilisateur de la table le demande.</span><span class="sxs-lookup"><span data-stu-id="0840c-106">Because none of the tables include multivalued properties in their default column sets, multivalued properties are included in a table only if the user of the table requests it.</span></span>

<span data-ttu-id="0840c-115">Un utilisateur de tableau demande le type d'affichage non défini par défaut en appelant la méthode [IMAPITable:: SetColumns](imapitable-setcolumns.md) avec l'indicateur  <span data-ttu-id="0840c-120">\*\*\*\* Toutefois, la propriété Restrict ne développe pas la colonne à valeurs multiples en lignes calculées supplémentaires.</span><span class="sxs-lookup"><span data-stu-id="0840c-120">**Restrict**, however, does not expand the multivalued column into additional computed rows.</span></span> <span data-ttu-id="0840c-121">Une colonne à valeurs multiples avec le jeu MVI_FLAG indique au fournisseur de services d'utiliser cette colonne dans le tableau.</span><span class="sxs-lookup"><span data-stu-id="0840c-121">A multivalued column with the MVI_FLAG set instructs the service provider to use that column in restricting the table.</span></span> <span data-ttu-id="0840c-122">S'il y a une valeur de propriété dans la restriction, il doit s'agir d'une balise de propriété à valeur unique identique à celle renvoyée par l'opération [IMAPITable:: QueryRows](imapitable-queryrows.md) pour la colonne.</span><span class="sxs-lookup"><span data-stu-id="0840c-122">If there is a property value in the restriction, it must be a single value property tag identical to the one that would be returned by [IMAPITable::QueryRows](imapitable-queryrows.md) for the column.</span></span> 
  
<span data-ttu-id="0840c-123">Les implémenteurs de tableau sont uniquement requis pour prendre en charge le type d'affichage par défaut et peuvent renvoyer la valeur MAPI_E_TOO_COMPLEX lorsqu'un appelant demande l'autre alternative.</span><span class="sxs-lookup"><span data-stu-id="0840c-123">Table implementers are only required to support the default type of display and can return the MAPI_E_TOO_COMPLEX value when a caller requests the other alternative.</span></span> <span data-ttu-id="0840c-124">La possibilité de prendre en charge les deux types d'affichage est essentielle pour les fournisseurs de banque de messages implémentant des tables de contenu de dossier.</span><span class="sxs-lookup"><span data-stu-id="0840c-124">The ability to support both types of display is most important for message store providers implementing folder contents tables.</span></span>
