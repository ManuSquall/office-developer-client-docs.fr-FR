<span data-ttu-id="4e719-105">Chaque message a une propriété de classe de message, **PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)), qui identifie le type, l'objectif ou le contenu du message.</span><span class="sxs-lookup"><span data-stu-id="4e719-105">Every message has a message class property, **PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)), which identifies the type, purpose, or content of the message.</span></span> <span data-ttu-id="4e719-106">**PR_MESSAGE_CLASS** est une propriété obligatoire pour tous les nouveaux messages.</span><span class="sxs-lookup"><span data-stu-id="4e719-106">**PR_MESSAGE_CLASS** is a required property on all new messages.</span></span> <span data-ttu-id="4e719-107">La classe d'un message détermine le formulaire utilisé pour présenter le message à l'utilisateur et au dossier de placement des messages entrants.</span><span class="sxs-lookup"><span data-stu-id="4e719-107">A message's class determines the form that is used to present the message to the user and the folder for placing incoming messages.</span></span> 
  
<span data-ttu-id="4e719-108">Les classes de message sont des chaînes de caractères qui contiennent des caractères ASCII 32 à 127 et qui sont délimitées par des points, mais elles ne peuvent pas se terminer par un point.</span><span class="sxs-lookup"><span data-stu-id="4e719-108">Message classes are case-sensitive character strings that contain ASCII characters 32 through 127 and are delimited by periods, but they cannot end with a period.</span></span> <span data-ttu-id="4e719-109">Chaque chaîne représente un niveau de sous-classe et il n'y a pas de limite au nombre de niveaux autorisés.</span><span class="sxs-lookup"><span data-stu-id="4e719-109">Each string represents a level of subclassing, and there is no limit to the number of levels allowed.</span></span> 
  
<span data-ttu-id="4e719-110">Par exemple, la plupart des messages que les applications clientes envoient et reçoivent entrent dans la classe de message **IPM** , une catégorie large qui décrit tous les messages interpersonnels (c'est-à-dire, les messages destinés à être lus par un utilisateur humain, et non par programme ordinateur).</span><span class="sxs-lookup"><span data-stu-id="4e719-110">For example, most messages that client applications send and receive fall into the **IPM** message class, a broad category that describes all interpersonal messages (that is, messages that are meant to be read by a human user, rather than programmatically by a computer).</span></span> <span data-ttu-id="4e719-111">Les fournisseurs de banques de messages décrivent plus précisément un message IPM en créant une sous-classe **IPM** .</span><span class="sxs-lookup"><span data-stu-id="4e719-111">Message store providers more precisely describe an IPM message by creating an **IPM** subclass.</span></span> <span data-ttu-id="4e719-112">La sous-classe **IPM** hérite des propriétés de la classe de message **IPM** .</span><span class="sxs-lookup"><span data-stu-id="4e719-112">The **IPM** subclass inherits the properties of the **IPM** message class.</span></span> <span data-ttu-id="4e719-113">Les sous-classes de la classe **IPM** sont nommées en concaténant d'autres chaînes de caractères sur l'identificateur IPM, comme **IPM. Remarque** pour décrire un message de note et **IPM. Contact** pour décrire un message de contact.</span><span class="sxs-lookup"><span data-stu-id="4e719-113">Subclasses of the **IPM** class are named by concatenating other character strings onto the IPM identifier, such as **IPM.Note** to describe a note message and **IPM.Contact** to describe a contact message.</span></span> 
  
<span data-ttu-id="4e719-114">Pour gérer l'affichage et la gestion des messages IPM, les clients peuvent utiliser un formulaire standard fourni par MAPI.</span><span class="sxs-lookup"><span data-stu-id="4e719-114">To handle the display and management of IPM messages, clients can use a standard form that MAPI provides.</span></span> <span data-ttu-id="4e719-115">Pour gérer l'affichage et la gestion des nouvelles classes de message, vous devez être développeur d'application cliente de deux options:</span><span class="sxs-lookup"><span data-stu-id="4e719-115">To handle the display and management of new message classes, you as a client application developer have two options:</span></span>
  
1. <span data-ttu-id="4e719-116">Vous pouvez créer un nouveau formulaire à l'aide de l'ensemble d'interfaces de formulaire défini par MAPI qu'un client standard peut utiliser.</span><span class="sxs-lookup"><span data-stu-id="4e719-116">You can create a new form by using the set of MAPI-defined form interfaces that a standard client can use.</span></span>
    
2. <span data-ttu-id="4e719-117">Vous pouvez écrire votre propre client en implémentant une application complète et autonome.</span><span class="sxs-lookup"><span data-stu-id="4e719-117">You can write your own client by implementing a complete, standalone application.</span></span> 
    
<span data-ttu-id="4e719-118">Bien que les clients doivent définir la propriété **PR_MESSAGE_CLASS** pour chaque message sortant sur une sous-classe de **IPM** ou **IPC**, le fournisseur de banque de messages a la responsabilité ultime de le définir.</span><span class="sxs-lookup"><span data-stu-id="4e719-118">Although clients should set the **PR_MESSAGE_CLASS** property for every outgoing message to a subclass of either **IPM** or **IPC**, the message store provider has the ultimate responsibility for setting it.</span></span> <span data-ttu-id="4e719-119">Par conséquent, si un client envoie un message sans définir sa classe de message, le fournisseur de banque de messages lui affecte la valeur par défaut appropriée pour le type de client approprié.</span><span class="sxs-lookup"><span data-stu-id="4e719-119">Therefore, if a client sends a message without setting its message class, the message store provider sets it to the appropriate default value for the appropriate type of client.</span></span> <span data-ttu-id="4e719-120">La classe de message par défaut pour les clients de messagerie interpersonnels est **IPM**; la classe de message par défaut pour les clients de communication interprocessus est **IPC**.</span><span class="sxs-lookup"><span data-stu-id="4e719-120">The default message class for interpersonal messaging clients is **IPM**; the default message class for interprocess communication clients is **IPC**.</span></span> 
  
<span data-ttu-id="4e719-121">Les classes de message ont une restriction de longueur de 255 caractères.</span><span class="sxs-lookup"><span data-stu-id="4e719-121">Message classes have a length restriction of 255 characters.</span></span> <span data-ttu-id="4e719-122">Toutefois, les classes de message ne doivent pas dépasser 127 caractères pour prendre en charge les classes de message utilisées dans les rapports.</span><span class="sxs-lookup"><span data-stu-id="4e719-122">However, message classes should not exceed 127 characters to support the message classes used in reports.</span></span> <span data-ttu-id="4e719-123">Les classes de message de rapport sont basées sur la classe du message d'origine, avec deux additions: un préfixe et un suffixe.</span><span class="sxs-lookup"><span data-stu-id="4e719-123">Report message classes are based on the class of the original message, with two additions: a prefix and a suffix.</span></span> <span data-ttu-id="4e719-124">Le rapport de préfixe indique que le message est un rapport et que le suffixe indique le type de rapport: DR (rapport de remise), notification d'échec de remise (rapport de non-remise), IPNRN (rapport de lecture) ou IPNNRN (rapport non lu).</span><span class="sxs-lookup"><span data-stu-id="4e719-124">The prefix REPORT indicates that the message is a report, and the suffix indicates the type of report: DR (delivery report), NDR (nondelivery report), IPNRN (read report), or IPNNRN (nonread report).</span></span> <span data-ttu-id="4e719-125">Notez que ces restrictions de longueur sont indiquées en caractères; sur les plateformes qui utilisent un jeu de caractères codés sur deux octets, le nombre d'octets réel peut être plus élevé.</span><span class="sxs-lookup"><span data-stu-id="4e719-125">Note that these length restrictions are given in characters; on platforms that use a double-byte character set, the actual byte count might be higher.</span></span> 
  
<span data-ttu-id="4e719-126">Les fournisseurs de banques de messages doivent renvoyer MAPI_E_INVALID_PARAMETER à partir de leurs implémentations de méthode [IMAPIProp:: SetProps](imapiprop-setprops.md) lorsqu'un client tente d'assigner une chaîne dépassant la limite autorisée pour sa classe de message.</span><span class="sxs-lookup"><span data-stu-id="4e719-126">Message store providers should return MAPI_E_INVALID_PARAMETER from their [IMAPIProp::SetProps](imapiprop-setprops.md) method implementations when a client attempts to assign a string that exceeds the allowable limit for their message class.</span></span>
