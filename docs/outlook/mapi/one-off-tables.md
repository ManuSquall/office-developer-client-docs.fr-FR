<span data-ttu-id="dabb3-109">Les clients créent de nouveaux destinataires :</span><span class="sxs-lookup"><span data-stu-id="dabb3-109">Clients create new recipients:</span></span>

- <span data-ttu-id="dabb3-111">Pour l’ajouter à l’un des conteneurs du carnet d’adresses.</span><span class="sxs-lookup"><span data-stu-id="dabb3-111">To add to one of the containers in the address book.</span></span>

<span data-ttu-id="dabb3-114">Lorsque le destinataire est inclus dans un message sortant, MAPI appelle la méthode [IABLogon::GetOneOffTable](iablogon-getoneofftable.md) du fournisseur de carnet d’adresses pour récupérer sa table.</span><span class="sxs-lookup"><span data-stu-id="dabb3-114">When the recipient will be included with an outgoing message, MAPI calls the address book provider's [IABLogon::GetOneOffTable](iablogon-getoneofftable.md) method to retrieve its one-off table.</span></span> <span data-ttu-id="dabb3-115">Le tableau unique inclut des modèles qui permettent à un utilisateur d’entrer des informations qui entraînent la création d’un destinataire avec une adresse valide.</span><span class="sxs-lookup"><span data-stu-id="dabb3-115">The one-off table includes templates which enable a user to enter information resulting in the creation of a recipient with a valid address.</span></span> <span data-ttu-id="dabb3-116">MAPI s’inscrit pour les notifications sur ce tableau, en le conservant ouvert afin que les modifications soient reflétées pour l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="dabb3-116">MAPI registers for notifications on this table, keeping it open so that changes can be reflected to the user.</span></span> <span data-ttu-id="dabb3-117">MAPI libère la table uniquement lorsque la méthode [IMAPIStatus::ValidateState](imapistatus-validatestate.md) de son sous-système ou de son objet d’état de carnet d’adresses est appelée.</span><span class="sxs-lookup"><span data-stu-id="dabb3-117">MAPI releases the table only when its subsystem or address book status object's [IMAPIStatus::ValidateState](imapistatus-validatestate.md) method is called.</span></span> 
  
<span data-ttu-id="dabb3-118">Lorsque le destinataire est ajouté à un conteneur, MAPI effectue un appel différent, appelant la méthode [IMAPIProp::OpenProperty](imapiprop-openproperty.md) du conteneur pour récupérer sa propriété **PR_CREATE_TEMPLATES** ([PidTagCreateTemplates](pidtagcreatetemplates-canonical-property.md)).</span><span class="sxs-lookup"><span data-stu-id="dabb3-118">When the recipient will be added to a container, MAPI makes a different call, invoking the container's [IMAPIProp::OpenProperty](imapiprop-openproperty.md) method to retrieve its **PR_CREATE_TEMPLATES** ([PidTagCreateTemplates](pidtagcreatetemplates-canonical-property.md)) property.</span></span> <span data-ttu-id="dabb3-119">L’ensemble de modèles inclus dans ce tableau unique représente les types de destinataires qui peuvent être ajoutés au conteneur.</span><span class="sxs-lookup"><span data-stu-id="dabb3-119">The set of templates included in this one-off table represents the types of recipients that can be added to the container.</span></span> <span data-ttu-id="dabb3-120">Par exemple, les serveurs de messagerie exposent souvent un conteneur pour chaque passerelle installée afin que chaque conteneur ne conserve que des adresses spécifiques à la passerelle correspondante.</span><span class="sxs-lookup"><span data-stu-id="dabb3-120">For example, mail servers often expose one container for every gateway that is installed so that each container only holds addresses specific to the corresponding gateway.</span></span>
