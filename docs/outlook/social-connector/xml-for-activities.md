<span data-ttu-id="90063-113">Dans le scénario suivant, le OSC synchronise dynamiquement les activités pour une personne sélectionnée dans le volet personnes, et obtient des détails sur cette personne:</span><span class="sxs-lookup"><span data-stu-id="90063-113">In the following scenario, the OSC dynamically synchronizes activities for a person selected in the People Pane, and gets details about that person:</span></span>
  
1. <span data-ttu-id="90063-114">Un fournisseur OSC qui prend en charge la synchronisation à la demande des activités indique qu'au OSC à l'aide des éléments **GetActivities** et **dynamicActivitiesLookupEx** .</span><span class="sxs-lookup"><span data-stu-id="90063-114">An OSC provider that supports on-demand synchronization of activities indicates that to the OSC by using the **getActivities** and **dynamicActivitiesLookupEx** elements.</span></span> <span data-ttu-id="90063-115">Le fournisseur OSC définit également l'élément **hashFunction** .</span><span class="sxs-lookup"><span data-stu-id="90063-115">The OSC provider also sets the **hashFunction** element.</span></span> <span data-ttu-id="90063-116">Les trois éléments sont des éléments enfants des **fonctionnalités**.</span><span class="sxs-lookup"><span data-stu-id="90063-116">All three elements are child elements of **capabilities**.</span></span>

4. <span data-ttu-id="90063-120">Le OSC actualise le volet personnes ou la carte de visite pour permettre à l'utilisateur de consulter les dernières activités de la personne sélectionnée.</span><span class="sxs-lookup"><span data-stu-id="90063-120">The OSC refreshes the People Pane or Contact Card to let the user see the latest activities of the selected person.</span></span> <span data-ttu-id="90063-121">Le OSC chiffre l'adresse SMTP de la personne à l'aide de la fonction de hachage spécifiée dans l'élément **hashFunction** , formant une chaîne XML conforme à la définition de schéma XML pour l'élément **hashedAddresses** .</span><span class="sxs-lookup"><span data-stu-id="90063-121">The OSC encrypts the person's SMTP address by using the hash function specified in the **hashFunction** element, forming an XML string that conforms to the XML schema definition for the **hashedAddresses** element.</span></span> 
    
5. <span data-ttu-id="90063-122">Le OSC appelle **ISocialSession2:: GetActivitiesEx**, en fournissant cette chaîne XML de l'adresse hachée en tant que paramètre _hashedAddresses_ , pour obtenir une collection d'activités en cours pour cette personne dans le paramètre _Activities_ .</span><span class="sxs-lookup"><span data-stu-id="90063-122">The OSC calls **ISocialSession2::GetActivitiesEx**, providing this XML string of the hashed address as the  _hashedAddresses_ parameter, to get a current collection of activities for that person in the  _activities_ parameter.</span></span> <span data-ttu-id="90063-123">La __ chaîne de paramètre Activities est conforme à la définition de schéma XML de l'élément **activityFeed** .</span><span class="sxs-lookup"><span data-stu-id="90063-123">The  _activities_ parameter string complies with the XML schema definition of the **activityFeed** element.</span></span> 
    
6. <span data-ttu-id="90063-124">En fonction de la définition de schéma XML pour **activityFeed**, le OSC analyse la chaîne d' _activités_ pour déterminer le type, la date de publication et d'autres informations sur chaque activité, et comment afficher l'activité.</span><span class="sxs-lookup"><span data-stu-id="90063-124">Based on the XML schema definition for **activityFeed**, the OSC further parses the  _activities_ string to find out the type, publish date, and other information about each activity, and how to display the activity.</span></span> 
    
7. <span data-ttu-id="90063-125">Pour obtenir des détails sur la personne sélectionnée, le OSC appelle [ISocialSession2:: GetPeopleDetails](isocialsession2-getpeopledetails.md), en fournissant la même chaîne XML d'adresses hachées que l'argument pour le paramètre _personsAddresses_ .</span><span class="sxs-lookup"><span data-stu-id="90063-125">To get details about the selected person, the OSC calls [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), providing the same XML string of hashed addresses as the argument for the  _personsAddresses_ parameter.</span></span> <span data-ttu-id="90063-126">Les détails relatifs à la personne sont renvoyés dans le paramètre _personsCollection_ .</span><span class="sxs-lookup"><span data-stu-id="90063-126">The details about the person are returned in the  _personsCollection_ parameter.</span></span> <span data-ttu-id="90063-127">Ces détails peuvent être les suivants: **prénom**, **nom**et **EmailAddress**.</span><span class="sxs-lookup"><span data-stu-id="90063-127">These details can include **firstName**, **lastName**, and **emailAddress**.</span></span> <span data-ttu-id="90063-128">Le paramètre _personsCollection_ est conforme à la définition de schéma XML pour l'élément **Person** .</span><span class="sxs-lookup"><span data-stu-id="90063-128">The  _personsCollection_ parameter conforms to the XML schema definition for the **person** element.</span></span>
