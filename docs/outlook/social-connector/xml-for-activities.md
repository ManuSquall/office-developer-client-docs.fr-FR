<span data-ttu-id="04d25-113">Dans le scénario suivant, OSC synchronise dynamiquement les activités d’une personne sélectionnée dans le volet Personnes et obtient des détails sur cette personne :</span><span class="sxs-lookup"><span data-stu-id="04d25-113">In the following scenario, the OSC dynamically synchronizes activities for a person selected in the People Pane, and gets details about that person:</span></span>
  
1. <span data-ttu-id="04d25-114">Un fournisseur OSC qui prend en charge la synchronisation à la demande des activités indique cela à l’OSC à l’aide des éléments **getActivities** et **dynamicActivitiesLookupEx.**</span><span class="sxs-lookup"><span data-stu-id="04d25-114">An OSC provider that supports on-demand synchronization of activities indicates that to the OSC by using the **getActivities** and **dynamicActivitiesLookupEx** elements.</span></span> <span data-ttu-id="04d25-115">Le fournisseur OSC définit également **l’élément hashFunction.**</span><span class="sxs-lookup"><span data-stu-id="04d25-115">The OSC provider also sets the **hashFunction** element.</span></span> <span data-ttu-id="04d25-116">Les trois éléments sont des éléments enfants **de fonctionnalités.**</span><span class="sxs-lookup"><span data-stu-id="04d25-116">All three elements are child elements of **capabilities**.</span></span> 
    
2. <span data-ttu-id="04d25-117">Le fournisseur OSC implémente les méthodes **ISocialProvider::GetCapabilities** et [ISocialSession2::GetActivitiesEx.](isocialsession2-getactivitiesex.md)</span><span class="sxs-lookup"><span data-stu-id="04d25-117">The OSC provider implements the **ISocialProvider::GetCapabilities** and [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) methods.</span></span> 
    
3. <span data-ttu-id="04d25-118">L’OSC appelle **ISocialProvider::GetCapabilities** pour vérifier la valeur de **getActivities** et **dynamicActivitiesLookupEx** afin de vérifier que le fournisseur OSC prend en charge la synchronisation à la demande des activités.</span><span class="sxs-lookup"><span data-stu-id="04d25-118">The OSC calls **ISocialProvider::GetCapabilities** to check the value of **getActivities** and **dynamicActivitiesLookupEx** to verify that the OSC provider supports on-demand synchronization of activities.</span></span> <span data-ttu-id="04d25-119">L’OSC note également la valeur de l’élément **hashFunction** pris en charge par le fournisseur OSC.</span><span class="sxs-lookup"><span data-stu-id="04d25-119">The OSC also notes the value of the **hashFunction** element supported by the OSC provider.</span></span> 
    
4. <span data-ttu-id="04d25-120">OsC actualise le volet Contacts ou la carte de visite pour que l’utilisateur voie les dernières activités de la personne sélectionnée.</span><span class="sxs-lookup"><span data-stu-id="04d25-120">The OSC refreshes the People Pane or Contact Card to let the user see the latest activities of the selected person.</span></span> <span data-ttu-id="04d25-121">OsC chiffre l’adresse SMTP de la personne à l’aide de la fonction de hachage spécifiée dans l’élément **hashFunction,** en formant une chaîne XML conforme à la définition de schéma XML de l’élément **hashedAddresses.**</span><span class="sxs-lookup"><span data-stu-id="04d25-121">The OSC encrypts the person's SMTP address by using the hash function specified in the **hashFunction** element, forming an XML string that conforms to the XML schema definition for the **hashedAddresses** element.</span></span> 
    
5. <span data-ttu-id="04d25-122">L’OSC appelle **ISocialSession2::GetActivitiesEx**, en fournissant cette chaîne XML de l’adresse hachée comme paramètre _hashedAddresses,_ pour obtenir une collection actuelle d’activités pour cette personne dans le paramètre d’activités. </span><span class="sxs-lookup"><span data-stu-id="04d25-122">The OSC calls **ISocialSession2::GetActivitiesEx**, providing this XML string of the hashed address as the  _hashedAddresses_ parameter, to get a current collection of activities for that person in the  _activities_ parameter.</span></span> <span data-ttu-id="04d25-123">La _chaîne du paramètre activities_ est conforme à la définition de schéma XML de l’élément **activityFeed.**</span><span class="sxs-lookup"><span data-stu-id="04d25-123">The  _activities_ parameter string complies with the XML schema definition of the **activityFeed** element.</span></span> 
    
6. <span data-ttu-id="04d25-124">En fonction de la définition de schéma XML pour **activityFeed**, l’OSC pare davantage la chaîne d’activités pour connaître le type, la date de publication et d’autres informations sur chaque activité et comment afficher l’activité. </span><span class="sxs-lookup"><span data-stu-id="04d25-124">Based on the XML schema definition for **activityFeed**, the OSC further parses the  _activities_ string to find out the type, publish date, and other information about each activity, and how to display the activity.</span></span> 
    
7. <span data-ttu-id="04d25-125">Pour obtenir des détails sur la personne sélectionnée, l’OSC appelle [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), fournissant la même chaîne XML d’adresses hachées que l’argument pour le paramètre _personsAddresses._</span><span class="sxs-lookup"><span data-stu-id="04d25-125">To get details about the selected person, the OSC calls [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), providing the same XML string of hashed addresses as the argument for the  _personsAddresses_ parameter.</span></span> <span data-ttu-id="04d25-126">Les détails sur la personne sont renvoyés dans le _paramètre personsCollection._</span><span class="sxs-lookup"><span data-stu-id="04d25-126">The details about the person are returned in the  _personsCollection_ parameter.</span></span> <span data-ttu-id="04d25-127">Ces détails peuvent inclure **firstName,** **lastName** et **emailAddress**.</span><span class="sxs-lookup"><span data-stu-id="04d25-127">These details can include **firstName**, **lastName**, and **emailAddress**.</span></span> <span data-ttu-id="04d25-128">Le  _paramètre personsCollection_ est conforme à la définition de schéma XML de l’élément **de** personne.</span><span class="sxs-lookup"><span data-stu-id="04d25-128">The  _personsCollection_ parameter conforms to the XML schema definition for the **person** element.</span></span>
