<span data-ttu-id="12637-113">Dans le scénario suivant, l’OSC dynamiquement synchronise les activités d’une personne sélectionnée dans le volet personnes et obtient des informations relatives à cette personne :</span><span class="sxs-lookup"><span data-stu-id="12637-113">In the following scenario, the OSC dynamically synchronizes activities for a person selected in the People Pane, and gets details about that person:</span></span>
  
1. <span data-ttu-id="12637-114">Un fournisseur OSC qui prend en charge la synchronisation de la demande d’activités qui indique à l’OSC en utilisant les éléments **getActivities** et **dynamicActivitiesLookupEx** .</span><span class="sxs-lookup"><span data-stu-id="12637-114">An OSC provider that supports on-demand synchronization of activities indicates that to the OSC by using the **getActivities** and **dynamicActivitiesLookupEx** elements.</span></span> <span data-ttu-id="12637-115">Le fournisseur OSC définit également l’élément **hashFunction** .</span><span class="sxs-lookup"><span data-stu-id="12637-115">The OSC provider also sets the **hashFunction** element.</span></span> <span data-ttu-id="12637-116">Tous les trois éléments sont des éléments enfants de **fonctionnalités**.</span><span class="sxs-lookup"><span data-stu-id="12637-116">All three elements are child elements of **capabilities**.</span></span> 
    
2. <span data-ttu-id="12637-117">Le fournisseur OSC implémente les méthodes **ISocialProvider::GetCapabilities** et [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) .</span><span class="sxs-lookup"><span data-stu-id="12637-117">The OSC provider implements the **ISocialProvider::GetCapabilities** and [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) methods.</span></span> 
    
3. <span data-ttu-id="12637-118">Les appels OSC **ISocialProvider::GetCapabilities** pour vérifier la valeur de **getActivities** et **dynamicActivitiesLookupEx** pour vérifier que le fournisseur OSC prend en charge la synchronisation de la demande d’activités.</span><span class="sxs-lookup"><span data-stu-id="12637-118">The OSC calls **ISocialProvider::GetCapabilities** to check the value of **getActivities** and **dynamicActivitiesLookupEx** to verify that the OSC provider supports on-demand synchronization of activities.</span></span> <span data-ttu-id="12637-119">L’OSC notes également la valeur de l’élément **hashFunction** pris en charge par le fournisseur OSC.</span><span class="sxs-lookup"><span data-stu-id="12637-119">The OSC also notes the value of the **hashFunction** element supported by the OSC provider.</span></span> 
    
4. <span data-ttu-id="12637-120">L’OSC actualise le volet personnes ou une carte de visite pour permettre à l’utilisateur de voir les activités le plus récent de la personne sélectionnée.</span><span class="sxs-lookup"><span data-stu-id="12637-120">The OSC refreshes the People Pane or Contact Card to let the user see the latest activities of the selected person.</span></span> <span data-ttu-id="12637-121">L’OSC chiffre adresse SMTP de la personne à l’aide de la fonction de hachage spécifiée dans l’élément **hashFunction** , formation d’une chaîne XML qui est conforme à la définition de schéma XML pour l’élément **hashedAddresses** .</span><span class="sxs-lookup"><span data-stu-id="12637-121">The OSC encrypts the person's SMTP address by using the hash function specified in the **hashFunction** element, forming an XML string that conforms to the XML schema definition for the **hashedAddresses** element.</span></span> 
    
5. <span data-ttu-id="12637-122">L’OSC appelle **ISocialSession2::GetActivitiesEx**, fournir cette chaîne XML de l’adresse de hachage en tant que paramètre _hashedAddresses_ , pour obtenir une collection d’activités en cours correspondant à cette personne dans le paramètre _activités_ .</span><span class="sxs-lookup"><span data-stu-id="12637-122">The OSC calls **ISocialSession2::GetActivitiesEx**, providing this XML string of the hashed address as the  _hashedAddresses_ parameter, to get a current collection of activities for that person in the  _activities_ parameter.</span></span> <span data-ttu-id="12637-123">La chaîne de paramètre _activités_ est conforme à la définition de schéma XML de l’élément **activityFeed** .</span><span class="sxs-lookup"><span data-stu-id="12637-123">The  _activities_ parameter string complies with the XML schema definition of the **activityFeed** element.</span></span> 
    
6. <span data-ttu-id="12637-124">En fonction de la définition de schéma XML pour **activityFeed**, plus le OSC analyse la chaîne de _activités_ pour déterminer le type, date et autres informations sur chaque activité et comment afficher l’activité de publication.</span><span class="sxs-lookup"><span data-stu-id="12637-124">Based on the XML schema definition for **activityFeed**, the OSC further parses the  _activities_ string to find out the type, publish date, and other information about each activity, and how to display the activity.</span></span> 
    
7. <span data-ttu-id="12637-125">Pour obtenir plus d’informations sur la personne sélectionnée, le OSC appelle [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), fournissant la même chaîne XML des adresses de hachage en tant qu’argument pour le paramètre _personsAddresses_ .</span><span class="sxs-lookup"><span data-stu-id="12637-125">To get details about the selected person, the OSC calls [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md), providing the same XML string of hashed addresses as the argument for the  _personsAddresses_ parameter.</span></span> <span data-ttu-id="12637-126">Les plus d’informations sur la personne sont retournés dans le paramètre _personsCollection_ .</span><span class="sxs-lookup"><span data-stu-id="12637-126">The details about the person are returned in the  _personsCollection_ parameter.</span></span> <span data-ttu-id="12637-127">Il peut s’agir **firstName**, **lastName**et **emailAddress**.</span><span class="sxs-lookup"><span data-stu-id="12637-127">These details can include **firstName**, **lastName**, and **emailAddress**.</span></span> <span data-ttu-id="12637-128">Le paramètre _personsCollection_ est conforme à la définition de schéma XML pour l’élément de la **personne** .</span><span class="sxs-lookup"><span data-stu-id="12637-128">The  _personsCollection_ parameter conforms to the XML schema definition for the **person** element.</span></span>
